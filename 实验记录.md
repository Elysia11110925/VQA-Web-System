## 三、模型方案

### 3.1 方案演变

| 阶段 | 方案 | 数据量 | 准确率 | 问题 |
|------|------|:--:|:--:|------|
| 初版 | 从头训练 Baseline | 473 条 | 26.85% | 只会答 yes/no |
| 扩数据 | 加图到 700 张，重训 | 3,779 条 | 27.82% | 提升微弱，仍倾向高频词 |
| **最终** | **ViLT 预训练模型** | 完整 VQA v2 | ~70% | 直接可用，回答多样 |

### 3.2 为什么从头训练效果差？

VQA 是 **1000 分类**问题，答案从 "yes" 到 "baseball" 到 "blue"。训练需要：
- 完整数据集：44 万条问题，8 万张图片
- 我们只有 3,779 条，平均每个答案类别出现不到 4 次
- 模型学不到足够的模式，退化到猜高频词（yes/no）

> **结论**：VQA 不适合用少量数据从头训练。正确做法是使用预训练模型（如 ViLT），就像第一周用 `bert-base-uncased` 一样。

### 3.3 ViLT 模型

| 参数 | 值 |
|------|-----|
| 模型名称 | `dandelin/vilt-b32-finetuned-vqa` |
| 架构 | Vision-and-Language Transformer |
| 预训练数据 | 完整 VQA v2.0 (44万+条) |
| 图片编码 | ViT (Vision Transformer) patch embedding |
| 文本编码 | BERT tokenizer |
| 融合方式 | Transformer 共注意力（co-attention） |
| 模型大小 | ~350MB |

## 五、模型架构（论文对照）

### 5.1 Baseline 架构（VqaModel）

```
图片 → VGG19 → FC → L2 Norm → img_feature (1024维)
问题 → Embedding → 2层LSTM → FC → qst_feature (1024维)
             ↓
    逐元素乘法融合 (img × qst)
             ↓
    2层 MLP + Dropout + Softmax
             ↓
    1000 类答案分布
```

### 5.2 SAN 架构（SANModel）

```
图片 → VGG19 → CNN特征 → 196个区域 (每区域512维)
问题 → LSTM → qst_feature (1024维)
             ↓
    Attention Layer 1: 用问题扫196个区域 → 找出大致相关区域 → u1
             ↓
    Attention Layer 2: 用u1再扫 → 精确定位到答案区域 → u2
             ↓
    MLP → 1000类答案
```

### 5.3 ViLT 架构（最终使用）

```
图片 → ViT Patch Embedding → image tokens
问题 → BERT Tokenizer → text tokens
             ↓
    拼接 [CLS] img_tokens [SEP] text_tokens [SEP]
             ↓
    多层 Transformer (共注意力)
             ↓
    [CLS] 输出 → 分类头 → 答案
```

## 六、训练过程记录

### 6.1 初版训练（473 样本）

| 阶段 | 观察 |
|------|------|
| Epoch 1-2 | loss 从 6.9 降至 4.6，val_acc ~25% |
| Epoch 3-5 | 最佳 val_acc 26.85% (Epoch 3) |
| 表现 | 对所有问题倾向回答 yes/no |

### 6.2 扩数据训练（3,779 样本）

| 阶段 | 观察 |
|------|------|
| Epoch 1 | Train 26.0%, Val 25.2% |
| Epoch 4 | **最佳 Val 27.82%**（后续过拟合） |
| Epoch 5-10 | Val 从 25.8% 降至 24.8%，Train 仍在 ~29% |
| 结论 | 数据量仍然不足，8 倍数据仅带来 1% 提升 |

### 6.3 训练超参数

| 参数 | Baseline | 说明 |
|------|:--:|------|
| embed_size | 1024 | 图文融合维度 |
| word_embed_size | 300 | 词向量维度 |
| hidden_size | 512 | LSTM 隐层 |
| num_layers | 2 | LSTM 层数 |
| learning_rate | 0.001 | Adam 优化器 |
| batch_size | 64 | GPU 8GB 显存限制 |
| num_epochs | 10 | Early stop 在 Epoch 4 |

## 八、问题与解决方案

| # | 问题 | 原因 | 解决方案 |
|---|------|------|----------|
| 1 | 从头训练只会答 yes/no | 训练数据太少（473条），1000分类 | 换用 ViLT 预训练模型 |
| 2 | 图片尺寸不一致导致 batch 失败 | COCO 图片尺寸各异 | 添加 `transforms.Resize((224,224))` |
| 3 | 数据路径指向 Google Drive | 原项目为 Colab 设计 | 重新生成 `.npy` 子集，修复为本地路径 |
| 4 | 验证集图片与训练集不重叠 | COCO train2014 ≠ val2014 | 分别下载两个集合的图片 |
| 5 | ViLT 下载走 huggingface.co 超时 | httpx 不走 HF_ENDPOINT 环境变量 | 使用 hf-mirror.com 镜像 + 设置 HF_HUB_ENDPOINT |
| 6 | 训练时 Python 输出被缓冲 | stdout 缓冲 | 使用 `py -u` 无缓冲模式 |

## 十、与第一周的对比

| 方面 | 第一周 ABSA | 第二周 VQA |
|------|-----------|-----------|
| 输入模态 | 文本 + 文本 | **图片 + 文本** |
| 模型 | LSTM + BERT | ViLT (Vision+Language Transformer) |
| 预训练 | GloVe 词向量 / BERT | VGG19 + ViLT |
| 数据量 | 2,328 条 (充足) | 需 44 万条 (严重不足) |
| 从头训练 | ✅ 可行 | ❌ 不可行 |
| Web 难点 | 文本输入很简单 | **文件上传 + 流处理** |
| 最终方案 | 自训练 BERT 78.7% | 预训练 ViLT ~70% |
| 后端通信 | JSON | **multipart/form-data** |

> **核心教训**：VQA 这类 1000 分类的多模态任务，必须依赖预训练模型。从头训练需要的数据量远超课程范围。这与第一周形成对比——NLP 的 3 分类任务在小数据上也能训练。
